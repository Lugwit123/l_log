# Toast显示时间调整

## 用户反馈
用户反馈toast能够显示，但是时间太短（3秒）。

## 调整内容

### 修改前
- **主Toast**: 3000毫秒（3秒）
- **备用Toast**: 3000毫秒（3秒）

### 修改后
- **主Toast**: 5000毫秒（5秒）
- **备用Toast**: 5000毫秒（5秒）

## 修改的文件

### 1. `code-block-ai.js`
```javascript
// 修改前
window.toast.success('已返回调用链视图');

// 修改后
window.toast.success('已返回调用链视图', 5000);
```

```javascript
// 修改前
}, 3000);

// 修改后
}, 5000);
```

### 2. `trace_tree_view.js`
```javascript
// 修改前
window.toast.success('已返回调用链视图');

// 修改后
window.toast.success('已返回调用链视图', 5000);
```

## 效果

现在点击"返回调用链"按钮后，toast通知会显示5秒钟，给用户足够的时间阅读通知内容。

## 考虑因素

### 为什么选择5秒？
- **可读性**: 5秒足够用户阅读和理解通知内容
- **不干扰**: 不会太长以至于影响后续操作
- **用户体验**: 提供充分的操作反馈时间

### 可选的调整
如果用户觉得5秒还是太短或太长，可以进一步调整：
- **更短**: 4000毫秒（4秒）
- **更长**: 6000毫秒（6秒）

## 测试验证

点击"返回调用链"按钮后：
1. ✅ Toast正常显示
2. ✅ 显示时间延长到5秒
3. ✅ 5秒后自动消失
4. ✅ 不影响其他功能

这个调整确保用户有足够的时间看到并理解toast通知的内容！
