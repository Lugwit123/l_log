# 后端与PC端分离启动说明

## ✅ 现在支持两种启动模式

### 模式1：一键启动（推荐新手）
**PC端自动启动后端**

```bash
cd D:\TD_Depot\Software\Lugwit_syncPlug\lugwit_insapp\trayapp\Lib\l_log\pcend
.\start.bat
```

**工作流程**：
1. PC端检查后端是否运行
2. 如果未运行，自动启动后端（后台线程）
3. 创建Qt原生窗口

**优点**：
- ✅ 简单，一键启动
- ✅ 适合日常使用

### 模式2：分离启动（推荐开发）
**后端和PC端独立运行**

#### 步骤1：启动后端服务
```bash
cd D:\TD_Depot\Software\Lugwit_syncPlug\lugwit_insapp\trayapp\Lib\l_log\backend
.\start_backend.bat
```

或直接运行Python：
```bash
cd D:\TD_Depot\Software\Lugwit_syncPlug\lugwit_insapp\trayapp\Lib\l_log\backend
D:\TD_Depot\Software\Lugwit_syncPlug\lugwit_insapp\python_env\lugwit_python.exe start_backend.py
```

**后端将运行在**：http://127.0.0.1:8000

#### 步骤2：启动PC端窗口
```bash
cd D:\TD_Depot\Software\Lugwit_syncPlug\lugwit_insapp\trayapp\Lib\l_log\pcend
.\start.bat
```

**PC端会**：
1. 检测到后端已运行
2. 直接连接到后端
3. 创建Qt原生窗口

**优点**：
- ✅ 后端独立运行，便于调试
- ✅ 可以重启PC端而不影响后端
- ✅ 多个客户端可以连接同一个后端
- ✅ 适合开发和测试

### 模式3：纯浏览器模式
**只启动后端，使用浏览器访问**

```bash
cd D:\TD_Depot\Software\Lugwit_syncPlug\lugwit_insapp\trayapp\Lib\l_log\backend
.\start_backend.bat
```

然后在浏览器中打开：http://127.0.0.1:8000

**优点**：
- ✅ 无需Qt依赖
- ✅ 跨平台（任何浏览器）
- ✅ 适合远程访问

## 🔧 智能启动逻辑

PC端 (`pcend/main.py`) 现在会：

1. **检查后端状态**（2秒超时）
   - 如果后端已运行 → 直接连接
   - 如果后端未运行 → 自动启动

2. **创建Qt窗口**
   - 使用Qt引擎
   - 连接到 http://127.0.0.1:8000

## 🎯 使用场景

### 场景1：日常使用
```bash
# 一键启动，PC端自动处理一切
pcend\start.bat
```

### 场景2：开发调试
```bash
# 终端1：启动后端（可以看到详细日志）
backend\start_backend.bat

# 终端2：启动PC端
pcend\start.bat
```

### 场景3：多客户端
```bash
# 终端1：启动后端
backend\start_backend.bat

# 终端2：PC端窗口
pcend\start.bat

# 浏览器：打开 http://127.0.0.1:8000
```

### 场景4：远程访问
```bash
# 服务器：启动后端（修改config.py中的HOST为0.0.0.0）
backend\start_backend.bat

# 客户端：浏览器访问 http://服务器IP:8000
```

## ⚙️ 配置

### 修改端口
编辑 `backend/config.py`：
```python
HOST = '127.0.0.1'  # 改为 '0.0.0.0' 允许远程访问
PORT = 8000         # 改为其他端口
```

### 修改PC端连接地址
编辑 `pcend/main.py`：
```python
HOST = '127.0.0.1'  # 后端地址
PORT = 8000         # 后端端口
```

## 📊 架构对比

### 之前（耦合）
```
PC端启动 → 强制启动后端 → 创建窗口
```

### 现在（分离）
```
方式1: PC端启动 → 检测后端 → 自动启动/连接 → 创建窗口
方式2: 后端启动 → PC端启动 → 检测并连接 → 创建窗口
方式3: 后端启动 → 浏览器访问
```

## ✅ 优势

1. **灵活性**：可以选择启动方式
2. **独立性**：后端和PC端可以独立重启
3. **可扩展性**：支持多客户端连接
4. **调试友好**：后端日志独立查看
5. **向后兼容**：原有的一键启动仍然可用

---

**推荐**：
- 日常使用 → `pcend\start.bat`（一键启动）
- 开发调试 → 分别启动后端和PC端
- 远程访问 → 只启动后端，浏览器访问
